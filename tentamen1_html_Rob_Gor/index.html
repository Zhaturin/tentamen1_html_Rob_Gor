Responsiv Webbdesign

Responsiv Webbdesign innebär att en webbsida ska vara tillgänglig på flertal olika platformer / devices. Detta är viktigt för att det finns multipla storlekar på skärmar, våra användare idag använder bland annat mobilen när de surfar och även surfplattor. Om man inte ha en responsiv sida så kommer användaren eventuellt ge upp när de kommer in på din webbsida.

Du kan göra webbsidan responsiv med hjälp av media queries, där man sätter storlekar på hur din sida ska se ut vid xx skärmstorlek. Man kan även bygga en webbsida direkt för mobil användning, dvs "Mobile first".

Brytpunkt innebär att vid en viss pixel så byter webbsidan layout.

Nätverk

Din dator skickar en förfrågan till din router och ifrågarsätter vart ligger "www.google.se", routerns skickar då tillbaka ett ack till din dator som berättar att den har fått din request. Efter detta så skickar din router vidare förfrågan till alla servers den är kopplad till, och får därefter tillbaka ett ack att den har fått förfrågan om vart ligger "www.google.se"?. Dessa servrar skickar sedan vidare samma förfrågan till nästa servers tills de hittar en server som har vet vart "www.google.se" ligger. Sedan skickar dessa servers tillbaka det till föregående server till föregående server och tillslut till din lokala router. Därefter skickar din router informationen om din förfrågan och du kan härmed ladda in sidan "www.google.se".

Bootstrap

Bootstrap är ett bibliotek för CSS, detta bibliotek används för att göra saker lättare, dvs de har förbestämda layouter som du kan anropa med rätt div.class

Man kan antingen ladda hem bootstrap och ha det i sin html map. Eller så kan man länka till bootstrap via en webbsida. Skillnaden mellan dessa 2 alternativ är att den du laddar ner förändrar sig aldrig i koden, medans om du länkar till ett onlien bibliotek så kommer koden förändra sig. Så när det kommer "major" updates till biblioteket så kan det göra att webbsida kanske inte fungerar 100%.
